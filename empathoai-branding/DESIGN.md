# Design System for EmpathoAI

> Category: Strategic Growth Architecture
> Premium diagnostic brand system. Built for slides, documents, web, and visual communication that feels controlled, analytical, and decisive.

## 1. Visual Theme & Atmosphere

EmpathoAI is not a startup aesthetic, a SaaS interface, or an agency moodboard. It is a **diagnostic system** expressed visually.

The atmosphere must feel:
- Controlled
- Precise
- Analytical
- Premium
- Tense in the right places
- Clear before persuasive

The visual rhythm should create the sensation that the viewer is being guided through a structured diagnosis rather than being sold a service with marketing tricks.

### Core Experience
- **System over showcase**: Every visual decision must support structure and interpretation.
- **Diagnosis before decoration**: Motion, contrast, and composition explain the system.
- **Authority over friendliness**: The tone is calm, sharp, and deliberate.
- **Decision over attention**: Orange is used to trigger decision, not to decorate.

### Emotional Signature
EmpathoAI should feel like:
- A controlled black room where complexity becomes legible
- A strategic instrument panel, not a startup landing page
- A premium advisory system with visual intelligence

It should **not** feel like:
- Generic agency branding
- Playful tech startup visuals
- Decorative futurism
- Template-based corporate slides
- Apple imitation without identity

## 2. Color Palette & Roles

Color is role-based, not decorative.

### Primary Structure
- **Structure Black** (`#0B0B0C`): primary background, dominant structural field
- **Surface Black** (`#1A1A1D`): cards, containers, overlays, raised surfaces
- **Divider Graphite** (`#2A2F36`): borders, separators, grid lines, rules

### Text & Clarity
- **Headline White** (`#F5F5F5`): hero headlines, primary statements, key conclusions
- **Secondary Gray** (`#7A8087`): labels, metadata, supportive copy
- **Body Gray** (`#8A8F98`): body copy, annotations, explanatory text

### Decision Accent
- **Decision Orange** (`#C56A1A`): insight triggers, key metrics, decision points, critical highlights

### Usage Rules
- Black controls structure
- White creates clarity
- Gray supports interpretation
- Orange signals decision

### Distribution Guidance
- Structure Black / Surface Black / white / grays should occupy **90–95%** of the visual system
- Decision Orange should occupy **5–10% max**

### Approved Pairings
- `#0B0B0C` + `#F5F5F5` → high-authority hero contrast
- `#1A1A1D` + `#8A8F98` → explanatory containers
- `#0B0B0C` + `#C56A1A` → decisive signal moments only
- `#F5F5F5` + `#C56A1A` on white documents → conclusion emphasis

### Prohibited Use
- No gradients
- No decorative accent colors
- No orange backgrounds for large sections
- No low-contrast gray-on-gray hierarchy failures
- No rainbow data visualization unless semantically required

## 3. Typography Rules

### Font Family
- **Primary / Display / Text:** `Helvetica Now`
- If unavailable for implementation previews, closest substitutes may be `Inter` for body and `Helvetica Neue` for display approximation, but official outputs should remain Helvetica Now.

### Weight System
- **700 Bold** → hero statements, slide headlines, key decisions
- **500 Medium** → section titles, subhead emphasis, structured labels
- **400 Regular** → body, explanatory copy, annotations
- **300 Light** → captions, metadata, tagline, low-emphasis support text

### Type Scale
| Role | Size | Weight | Line Height | Tracking | Use |
|------|------|--------|-------------|----------|-----|
| Hero XXL | 72px | 700 | 0.95-1.00 | -0.04em | Opening statements, high-impact covers |
| Hero XL | 56px | 700 | 1.00-1.04 | -0.03em | Main slide headlines |
| Display L | 44px | 700 | 1.05 | -0.02em | Chapter transitions, architecture frames |
| Display M | 36px | 700 | 1.08 | -0.02em | Core section headings |
| Section Title | 28px | 500 | 1.12 | -0.01em | Modules, sub-sections |
| Subhead | 22px | 500 | 1.18 | 0 | Supporting thesis lines |
| Body Large | 18px | 400 | 1.45 | 0 | Short explanation blocks |
| Body | 16px | 400 | 1.50 | 0 | Standard body copy |
| Label | 13px | 500 | 1.35 | 0.08em | System labels, eyebrow text |
| Meta | 12px | 300-400 | 1.35 | 0.10em | Captions, notes, supporting detail |

### Typography Principles
- Headlines are declarative, never fluffy.
- Body copy should be short, segmented, and scannable.
- Avoid dense paragraphs; prefer 1–3 line units.
- Use oversized headlines to create controlled tension.
- Use light labels to frame analysis, not to decorate.

## 4. Spacing, Grid & Composition

### Spacing Scale
- `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96`

### Grid Principles
- Default to **left alignment**
- Use strong vertical rhythm
- Favor large margins and negative space
- Build compositions from clear blocks before adding detail

### Composition Rules
- One dominant idea per slide
- One dominant headline per section
- Allow **controlled asymmetry**
- Use offset blocks, layered sections, and isolated triggers
- Favor structural segmentation over decorative containers

### Layout Tension Model
- **Symmetry** = safety
- **Asymmetry** = tension
- **Tension** = perception

Use asymmetry selectively to make the viewer feel guided into a diagnosis.

### Negative Space
Negative space is not emptiness. It is a reasoning tool.
Use it to:
- isolate insight
- slow the viewer down
- separate diagnosis from implication
- create premium calm

## 5. Slide System

EmpathoAI slides are not dense decks. They are structured arguments.

### Universal Slide Rules
- One idea per slide
- Dark background by default: `#0B0B0C`
- Left-aligned core content
- Minimal text, but not vague text
- Orange used only for the key insight, variable, or decision
- Every slide must have a clear rhetorical job

### Recommended Slide Sequence for Commercial / Strategic Decks
1. **Disrupt Belief** — challenge the client’s current interpretation
2. **Reveal Problem** — explain what is actually broken
3. **Expose System** — show the structure beneath the visible problem
4. **Map Consequence** — quantify or frame the cost of inaction
5. **Reframe Opportunity** — introduce the strategic opening
6. **Present Architecture** — show EmpathoAI’s system / method
7. **Prove Logic** — evidence, case logic, diagnostics, benchmarks
8. **Define Offer** — scope, engagement model, deliverables
9. **Drive Decision** — next step, decision framing, CTA

### Slide Archetypes
- **Hero Insight Slide**: oversized statement + sparse support line
- **Diagnostic Breakdown Slide**: segmented variables + annotations
- **System Architecture Slide**: structured framework, layered components
- **Consequence Slide**: metric, comparison, or before/after logic
- **Offer Slide**: service model with disciplined hierarchy
- **Decision Slide**: one direct CTA with minimal distraction

### Slide Copy Pattern
Use this order whenever possible:
1. Insight
2. System explanation
3. Decision

### Example Tone
Wrong:
- “We help businesses grow with AI.”

Correct:
- “Your growth is not failing.
Your system is.”

## 6. Component Language

### Headlines
- Large, direct, statement-driven
- Avoid filler words and soft lead-ins
- Use contrast between a dominant white statement and a restrained gray support line

### Eyebrows / Labels
- Light or medium weight
- Uppercase or high-tracking style
- Functional, not decorative
- Example: `DIAGNOSIS`, `SYSTEM FAILURE`, `DECISION WINDOW`

### Metric Blocks
- Large number or short metric label
- Minimal framing
- Orange only when the metric is decisive
- Avoid dashboard clutter

### Dividers & Rules
- Use `#2A2F36`
- Thin, intentional, structural
- Divide logic, not just space

### Containers
- Use `#1A1A1D` when a concept needs containment
- Moderate radius: `12px` to `20px`
- Avoid excessive cardification

### Diagrams
Preferred visual language:
- segmented systems
- annotated flows
- causal chains
- layered architecture
- diagnostic maps

Avoid:
- decorative blobs
- random icons
- ornamental infographics

### CTA Blocks
- One CTA only
- No CTA clusters
- Clear decision framing
- Orange may be used to signal action, but not as a full-screen treatment

## 7. Motion & Interaction

Motion must explain the system, not entertain.

### Allowed Motion
- Fade
- Directional slide
- Sequential reveal
- Mask reveal
- Layered scroll / parallax only when it explains depth or causality
- Section transitions with clear purpose

### Timing Guidance
- Fast enough to feel controlled
- Slow enough to feel intentional
- Avoid elastic or playful easing

### Motion Principles
- Reveal hierarchy over time
- Use stagger to show sequence or logic
- Use transitions to move from symptom → system → decision

### Forbidden Motion
- Decorative animation
- Floating objects without meaning
- Bouncy easing
- Excessive blur / glow effects
- Motion that competes with the message

## 8. Contrast & Rhythm

Premium EmpathoAI output needs contrast moments.

### Use Contrast Through
- dark-to-light transitions between sections when needed
- oversized type followed by compressed detail
- isolated orange triggers in otherwise restrained compositions
- large voids against dense diagnostic clusters

### Rhythm Rule
If everything is equally disciplined, nothing feels important.
Build controlled variation into the narrative.

## 9. Documents & Light-Mode Adaptation

EmpathoAI documents may use light mode, but must preserve the same system logic.

### Document Palette
- Background: `#FFFFFF`
- Headline: `#0B0B0C`
- Body: `#7A8087` / `#8A8F98`
- Accent / conclusion: `#C56A1A`

### Document Structure
1. Insight
2. Diagnosis
3. Architecture
4. Execution

### Document Rules
- Maintain generous whitespace
- Use black for structure, not visual heaviness
- Orange highlights conclusions, not decoration
- Preserve left alignment and analytical clarity

## 10. Data Visualization

Charts and visual evidence should feel analytical, not startup-like.

### Preferred Style
- dark background charts for slides
- thin grid lines
- minimal axis styling
- one primary data series when possible
- orange only for a decisive metric, inflection point, or comparison signal

### Preferred Chart Types
- line charts
- bar comparisons
- segmented funnels
- stage-based diagrams
- benchmark comparisons

### Avoid
- colorful multi-series charts without narrative focus
- pie charts unless unavoidable
- visual noise from unnecessary labels or legends

## 11. Voice & Copy

### Tone
- Direct
- Structured
- Calm
- Sharp
- Strategic

### Writing Rules
- Statements over paragraphs
- No fluff
- No startup optimism clichés
- No “we are passionate about…” phrasing
- No vague innovation language

### Preferred Sentence Behavior
- short
- declarative
- causal
- diagnostic

### Copy Framework
- Disrupt belief
- Reveal structure
- Name implication
- Force decision

## 12. Anti-Patterns

The system has failed if the output feels like:
- an agency sales deck
- a SaaS template
- a startup pitch aesthetic
- an over-designed design portfolio
- a direct Apple imitation
- a generic corporate presentation with black background

### Specific Anti-Patterns
- too many bullets
- too many cards
- too much orange
- centered everything
- decorative gradients
- friendly or playful iconography
- excessive motion
- generic stock visual metaphors
- no diagnostic logic in the composition

## 13. Responsive & Presentation Behavior

### Desktop / Presentation Mode
- Use large left-anchored statements
- Create deliberate pacing between dense and sparse slides
- Use full-screen compositions with strong margin discipline

### Mobile / Narrow View Adaptation
- Preserve one idea per frame
- Stack diagnostic blocks vertically
- Maintain headline dominance
- Remove non-essential supporting detail before shrinking typography too aggressively

## 14. Agent Prompt Guide

Use prompts that specify both the rhetorical job and the system behavior.

### Good Prompt Pattern
> Create a premium EmpathoAI commercial proposal deck.
> Use the EmpathoAI design system: dark diagnostic aesthetic, left-aligned hierarchy, restrained typography, and orange only for decision triggers.
> The deck should feel premium like top-tier editorial/product brands, but remain analytical, structured, and unmistakably EmpathoAI.
> Build the narrative as: disrupt belief -> diagnosis -> system -> consequence -> architecture -> offer -> decision.

### Good Presentation Requests
- “Create a 10-slide commercial proposal using the EmpathoAI system.”
- “Make this deck feel more diagnostic and less agency-like.”
- “Reduce decorative elements and increase structural tension.”
- “Rework this presentation so it feels premium, analytical, and decision-oriented.”

### Prompt Keywords That Match the System
- premium
- diagnostic
- structured
- strategic
- controlled
- analytical
- decisive
- left-aligned
- minimal but tense

## 15. Source Identity Rule

EmpathoAI output should always feel like:

> A controlled system that reveals structure, sharpens diagnosis, and forces clarity before action.

If an output is beautiful but does not create that sensation, it is off-brand.
